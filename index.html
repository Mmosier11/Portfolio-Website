TeamBandit: Teams Manager

About the Project

For my final year in college I was assigned a team to work on a client project for my Capstone. We were tasked with creating TeamBandit, a web application whose goal was to streamline course and team management. This was primarily for my Capstone course but there was possibilities to move beyond.

Built With:

PostgreSQL, Express.js, React, and Node.js

GitHub Link

https://github.com/eckdoerry/TeamBandit

My Contributions

My contributions to this project focused heavily on building the foundation for this application. This included:

  • Building the applications web page routing
  • Adding Different Users (Instructor/Student/Mentor)
  • Building our Express.js Routes
  • Building the React code/folder architecture
  • Focusing on UI/UX for Application

Organizer Search Feature

While not originally planned for, this feature was adding towards the end of TeamBandit's development. With this feature you can search through the entirety of TeamBandit's Organizers dynamically populating the output field. This gives users who don't have accounts the ability to see courses and projects.

User Creation

User creation was an integral part of our application. The different user types include:

  • Students
  • Organizers
  • Mentors

While Mentors aren't implemented yet, the foundations for their implementation are there. Students can't be created unless they have been added to an Organizer's course.

Course Overview

Courses are the key to course management. Here in the course overview page you can get an overview of all of your courses as well as some statistics such as:

  • Course Organizer
  • Course Semester
  • Total Projects
  • Total Students
  • Date Created

Client List

For the Capstone course, it is crucial for clients to propose projects that student teams can work on. The client list makes it possible to add and manage clients.

Client Email Overview

With managing one course being a small part of ones college career, its hard to check for specific emails from clients in regards to specific courses. We designed a way to grab the emails regarding the specific course and display them in the application.

Client Email View

Another feature we implemented in regards to the client's emails was being able to see relevant data in regards to specific emails. This includes:

  • Sender
  • Subject
  • Date Sent
  • Body of Email
  • Any Attachments

CSV Uploader

Our client for this project already had list of students and clients ready for use in this application. In order to make the transition easier we created a CSV uploader. You could search for CSVs on your PC to upload or Drag and Drop them in.

Student List

Like the client list there was important information to display for the students. This included:

  • First and Last Name
  • Student ID
  • Student Email
  • Student GPA

Adding Information

Besides CSVs we added modals to make it easier to add any information. When relevant like for adding clients to a project we made it be a drop down menu.

Project Preferences

At the beginning of a course, students were able to submit their project preferences letting the course organizer know what projects they were interested in.

Team Assignment

Once all of the projects have been added and students have submitted their project preferences the organizer is now able to assign students to teams with a very intuitive 'click to add' system. This system provides:

  • Color Coding
  • Student Statistics
  • Project Statistics

Project Lead Assignment

Once students are added to a project the organizer has the capability to designate a student as the Team Leader. This will give them access to features they will need to run their team.

Team Lead Actions

Team Leads are integral to filling out information for their team. This includes:

  • Team Name
  • Team Logo
  • Team Assignments
  • Information about the Team
  • Website Information

Pre-Generated Website

From the details given by the Team lead. We provided Pre-Generated team websites. These websites were based on a format the client pre-determined. Once the information is filled out the corresponding website is created.

Project Timeline

Ultimately we had 16 weeks to create this project. Here you can see our project timeline we utilized when creating the product. We followed a more agile practice when determining what to accomplish for the weeks, but this timeline is what we followed.

End Result

At the end of the 16 weeks, we were able to accomplish our clients MVP. This features include:

  • Projects List
  • Schedule
  • Student List
  • Assignments
  • Mentor List
  • Course Settings
  • Student Assignment Submission
  • Team Lead Actions
  • Pre-Generated Website
